## 1. 内部ロジック設計 (REQ-1.2 準拠)

### 1.1 価格計算エンジン
- **計算ロジック**:
  - `BasePrice`: `products.base_price`
  - `SpecAdjust`: `Σ(product_specs.price_modifier)` (選択された仕様の合計)
  - `LengthAdjust`: `Math.floor(length / 100) * 500` (100mm ごとに 500円)
  - **ボリュームディスカウント**: `qty >= 10 ? 0.95 : qty >= 50 ? 0.90 : 1.0` (5個以上5%引、50個以上10%引)
  - **計算式**: `(BasePrice + SpecAdjust + LengthAdjust) * Qty * Discount * 1.10` (税込)

### 1.2 SVG リアルタイム描画 (REQ-1.2)
- **座標計算**:
  - `ViewBox`: `0 0 800 200`
  - `RollerRect`: 
    - `x`: `(800 - w) / 2`
    - `width (w)`: `300 + (length / 5)` px
    - `height (h)`: `diameter === 'φ50' ? 60 : 40` px
  - `Color`: 材質ステートに応じて `#BBBBBB`（スチール）と `#E0E0E0`（ステンレス）を切り替え。

### 1.3 PDF 見積書生成 (REQ-1.3)
- **ライブラリ**: `pdf-lib`
- **生成プロセス**:
  1. `CartContext` の `cart` 配列を走査。
  2. 商品ごとに「ベース名 + 仕様文字列」を一行の明細としてフォーマット。
  3. `y` 座標を管理し、1明細ごとに 20pt ずつ減算して描画。
  4. 生成した `Uint8Array` を Blob 経由でダウンロード実行。

## 2. データ構造設計 (REQ-3.1/3.2)

### 2.1 データベーステーブル (SQLite3/PostgreSQL)
- **`companies`**: `id`, `name`, `billing_type` ('credit', 'invoice')
- **`users`**: `id`, `company_id`, `email`, `password_hash`, `role` ('admin', 'approver', 'orderer', 'designer')
- **`products`**: `id`, `category`, `base_name`, `base_price`, `description`
- **`product_specs`**: `id`, `product_id`, `name`, `value`, `price_modifier`
- **`orders`**: `id`, `user_id`, `company_id`, `total_price`, `status`, `created_at`
- **`order_items`**: `id`, `order_id`, `product_id`, `specs_json`, `quantity`, `unit_price`

## 3. API 設計

### 3.1 認証・セッション
- `POST /api/auth/login`: ログイン認証（JWT 発行）
- `GET /api/auth/me`: ログイン中ユーザー情報の取得

### 3.2 商品・シミュレーション
- `GET /api/products`: 商品一覧
- `GET /api/products/:id`: 商品詳細と仕様マスタ
- `POST /api/simulate/price`: 仕様に基づいた確定価格算出（サーバーサイドバリデーション含む）

### 3.3 注文・承認
- `POST /api/orders`: 注文申請/発注の登録
- `GET /api/orders/:id`: 注文詳細の取得
- `PATCH /api/orders/:id/approve`: 注文承認（承認者権限が必要）

---

## 4. セキュリティ・運用実装

- **認証ミドルウェア**: Express のミドルウェアにより、各 API リクエストの JWT 有効性を検証。
- **認可制御**: ユーザーの `role` に応じた権限チェック（例: `orders/:id/approve` は `approver` のみ許可）。
- **入力バリデーション**: `express-validator` を使用し、すべての POST パラメータ（数値範囲、型）を厳密にチェック。

### 2.2 アプリケーション・ステート (CartContext)
- `CartItem` 型:
  ```typescript
  {
    cartId: string;      // ユニークID (仕様ごとに別アイテム)
    id: number;          // 商品ID
    base_name: string;
    specs: { [key: string]: string }; // {'diameter': 'φ30', 'material': 'SUS'}
    length: number;      // 選択された長さ
    price: number;       // 計算済み単価
    quantity: number;    // 注文数
  }
  ```

## 3. 非機能要件の実装手法 (REQ-2.1/2.2)

- **高速更新**: React の `useEffect` による依存配列（`selectedSpecs`, `length`）の監視により、再レンダリングを最小限に抑えつつ即時反映。
- **モバイル対応**: CSS メディアクエリを使用し、詳細画面の左右 2 カラムを縦 1 カラムにスタック。
