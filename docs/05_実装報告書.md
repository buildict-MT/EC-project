# 実装報告書：RecruitDirect プロジェクト

## 1. 概要
本報告書は、製造部品特化型 B2B EC サイト「RecruitDirect」における一連の実装および設計変更作業を記録したものである。旧名称「MonoDirect」からのリブランド、および B2B 実運用に耐えうる高度な機能設計と実装を完了した。

## 2. 実施内容

### 2.1 プロジェクト・リブランディング
- **プロジェクト名変更**: 全ソースコード、ドキュメント（README、設計書）、UI（ヘッダー、フッター、ブラウザタイトル）における「MonoDirect」を「RecruitDirect」へ統一。
- **リポジトリ管理**: GitHub リポジトリ (`https://github.com/buildict-MT/EC-project`) への初期登録および継続的なプッシュを完了。

### 2.2 設計の拡張（B2B 特化要件）
既存の設計書を大幅に拡張し、以下の定義を追加・統合した：
- **組織・権限設計**: 企業・部署単位の管理、および4つのロール（管理者/承認者/発注担当/設計担当）の定義。
- **承認ワークフロー**: 発注担当者による「承認申請」から承認者による「確定」までのフロー。
- **価格戦略**: 数量に応じたボリュームディスカウント（10個以上で5%引、50個以上で10%引）のロジック。
- **システム連携**: 基幹システム（ERP）連携および CAD データ動的生成のアーキテクチャ設計。

### 2.3 バックエンド実装 (Node.js / Express / SQLite)
- **データベース構築**: SQLite を使用し、B2B に必要な 6 つの主要テーブル（companies, users, products, specs, orders, order_items）を構築。
- **認証システム**: `jsonwebtoken` (JWT) と `bcryptjs` (パスワードハッシュ化) を使用したセキュアなログイン API を実装。
- **注文 API**: ユーザーのロールに応じた注文ステータス管理（pending_approval / approved）および、トランザクションを考慮した注文登録 API を実装。

### 2.4 フロントエンド実装 (React / TypeScript)
- **認証状態管理**: `AuthContext` を新規作成し、アプリ全体でのログイン状態とユーザー情報の永続化を実現。
- **新規画面作成**:
    - **ログイン画面**: 企業 ID、ユーザー ID による認証インターフェース。
    - **マイページ（アカウント）画面**: ユーザープロフィールの表示および、過去の注文・申請履歴の照会機能。
- **シミュレーター強化**:
    - ボリュームディスカウントロジックを搭載。数量変更に合わせて割引率と単価、合計金額が即時計算される UI を実装。
- **カート・注文機能**:
    - ロールに基づいた「承認を申請する」または「注文を確定する」ボタンの動的切り替え。
    - PDF 見積書生成機能における企業名の動的反映とブランド名の更新。

## 3. 使用技術スタック
- **Frontend**: React (Hooks, Context API), React Router, Lucide-React, pdf-lib
- **Backend**: Express.js, SQLite3, jsonwebtoken, bcryptjs, cors
- **Tools**: Git, GitHub, npm, TypeScript
